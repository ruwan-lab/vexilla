# Data model (SQLite)

This schema is the **contract** between components. Do not deviate without updating
this file. Two databases:

- **Runtime DB** (`/var/lib/vexilla/vexilla.db`) — read/write, holds observations.
- **Knowledge base DB** (`/usr/share/vexilla/kb.db`) — read-only, shipped. See
  `knowledge-base.md` for its schema.

All timestamps are UTC epoch seconds (INTEGER). SQLite in **WAL** mode.

## Runtime DB schema

```sql
-- Applications observed on the device.
CREATE TABLE app (
    id           INTEGER PRIMARY KEY,
    name         TEXT NOT NULL,           -- e.g. "firefox"
    exe_path     TEXT,                    -- resolved executable path
    first_seen   INTEGER NOT NULL,
    last_seen    INTEGER NOT NULL,
    UNIQUE(name, exe_path)
);

-- Remote endpoints, named where possible.
CREATE TABLE endpoint (
    id           INTEGER PRIMARY KEY,
    ip           TEXT NOT NULL,           -- remote IP (v4 or v6)
    domain       TEXT,                    -- resolved domain, NULL if unknown
    name_source  TEXT,                    -- 'dns' | 'reverse' | 'none'
    first_seen   INTEGER NOT NULL,
    last_seen    INTEGER NOT NULL,
    UNIQUE(ip, domain)
);

-- Individual observed connections (flows). One row per (app, endpoint, port, proto)
-- session window; byte counts are cumulative for that window.
CREATE TABLE flow (
    id             INTEGER PRIMARY KEY,
    app_id         INTEGER NOT NULL REFERENCES app(id),
    endpoint_id    INTEGER NOT NULL REFERENCES endpoint(id),
    protocol       TEXT NOT NULL,          -- 'tcp' | 'udp'
    remote_port    INTEGER NOT NULL,
    bytes_sent     INTEGER NOT NULL DEFAULT 0,
    bytes_recv     INTEGER NOT NULL DEFAULT 0,
    is_background  INTEGER NOT NULL DEFAULT 0,  -- 0 active, 1 background
    started_at     INTEGER NOT NULL,
    last_seen      INTEGER NOT NULL
);
CREATE INDEX idx_flow_app   ON flow(app_id, last_seen);
CREATE INDEX idx_flow_ep    ON flow(endpoint_id, last_seen);
CREATE INDEX idx_flow_time  ON flow(last_seen);

-- Passive DNS cache: IP -> domain observations from captured DNS responses.
CREATE TABLE dns_cache (
    id           INTEGER PRIMARY KEY,
    ip           TEXT NOT NULL,
    domain       TEXT NOT NULL,
    observed_at  INTEGER NOT NULL,
    ttl          INTEGER,
    UNIQUE(ip, domain)
);
CREATE INDEX idx_dns_ip ON dns_cache(ip);

-- Hourly aggregates for fast dashboards (rolled up by the insight engine).
CREATE TABLE agg_hourly (
    id            INTEGER PRIMARY KEY,
    hour_start    INTEGER NOT NULL,        -- epoch, truncated to the hour
    app_id        INTEGER REFERENCES app(id),
    endpoint_id   INTEGER REFERENCES endpoint(id),
    bytes_sent    INTEGER NOT NULL DEFAULT 0,
    bytes_recv    INTEGER NOT NULL DEFAULT 0,
    conn_count    INTEGER NOT NULL DEFAULT 0,
    bg_bytes      INTEGER NOT NULL DEFAULT 0,  -- of the above, how much was background
    UNIQUE(hour_start, app_id, endpoint_id)
);
CREATE INDEX idx_agg_hour ON agg_hourly(hour_start);

-- Insights: flagged items produced by heuristics. Human-readable text lives here.
CREATE TABLE insight (
    id            INTEGER PRIMARY KEY,
    kind          TEXT NOT NULL,           -- see insight-engine.md (new_domain, tracker, ...)
    severity      TEXT NOT NULL,           -- 'info' | 'notice' | 'warning'
    app_id        INTEGER REFERENCES app(id),
    endpoint_id   INTEGER REFERENCES endpoint(id),
    title         TEXT NOT NULL,           -- plain-language, e.g. "New service contacted"
    body          TEXT NOT NULL,           -- plain-language explanation
    suggestion    TEXT,                    -- plain-language action, nullable
    evidence      TEXT,                    -- JSON blob of raw numbers backing the insight
    created_at    INTEGER NOT NULL,
    dismissed     INTEGER NOT NULL DEFAULT 0
);
CREATE INDEX idx_insight_created ON insight(created_at, dismissed);

-- Cached daily plain-language summary (regenerated by the insight engine).
CREATE TABLE summary (
    id            INTEGER PRIMARY KEY,
    day_start     INTEGER NOT NULL UNIQUE, -- epoch, truncated to the day
    text          TEXT NOT NULL,           -- the plain-language summary
    stats_json    TEXT NOT NULL,           -- structured numbers used to render it
    generated_at  INTEGER NOT NULL
);

-- Key/value settings. All keys have code defaults; this table only stores overrides.
CREATE TABLE setting (
    key    TEXT PRIMARY KEY,
    value  TEXT NOT NULL
);

-- Schema version for migrations.
CREATE TABLE schema_meta (
    key    TEXT PRIMARY KEY,
    value  TEXT NOT NULL
);
INSERT INTO schema_meta(key, value) VALUES ('version', '1');
```

## Notes on modeling decisions

- **`flow` is windowed, not per-packet.** The collector updates an existing open flow
  row's byte counters/`last_seen` rather than inserting per packet — keeps the DB small.
- **`agg_hourly` powers the dashboard.** Never scan `flow` for UI reads; read aggregates.
- **`endpoint.domain` may be NULL.** Naming can lag or fail (encrypted DNS, direct IP).
  UI must show a friendly fallback ("unknown service at 1.2.3.4").
- **`insight.evidence`** stores the raw numbers so any claim is auditable/explainable
  (supports the "explainable, not magic" principle).
- **KB is never joined at write time** by the collector; only the insight engine and API
  read `kb.db` (attach as a separate connection or `ATTACH DATABASE`).

## Retention

Default 30 days (`setting` key `retention_days`). A daily prune job deletes `flow`
and `agg_hourly` rows older than the window; `summary` and `insight` may be kept longer
(configurable). See `mvp-scope.md` for the prune task.
